# Object-Oriented Blackjack Simulator

A terminal-based, object-oriented Blackjack (21) game simulator built to demonstrate clean software design, state tracking, and foundational algorithmic logic.

## 🚀 Features
- **Object-Oriented Architecture:** Formatted with modular encapsulation to separate game loops, card deck management, and participant hand states.
- **Dynamic State Tracking:** Real-time evaluation of player choices (Hit/Stand), dealer artificial intelligence constraints (hitting on soft 17s), and score calculation.
- **Robust Input Validation:** Built-in error handling to gracefully manage invalid terminal choices or unexpected runtime inputs.

## 🛠️ Concepts Demonstrated
This project serves as a practical application of core paradigms covered in **Computer Science 1 (CS1)** and **Processes of Object-Oriented Software Design (POOSD)**:
- **Encapsulation:** Isolating properties and behaviors within specialized classes (`Deck`, `Hand`, `GameEngine`).
- **Control Flow & Conditional Logic:** Complex conditional statements governing win/loss conditions, tie scenarios (pushes), and natural Blackjacks.
- **Algorithmic Randomization:** Utilizing secure randomization logic to handle standard deck shuffling.
